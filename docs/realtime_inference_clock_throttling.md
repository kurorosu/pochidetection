# リアルタイム推論時の GPU クロックスロットリング調査

関連 Issue: #374

## 概要

Webcam リアルタイム推論で, 開始後数秒間は画像推論と同等の速度が出るが, その後推論速度が低下し, 一定時間後に再び回復するサイクルを繰り返す現象を調査した. 原因は GPU の Power Limit によるクロックスロットリングであり, ソフトウェア側の不具合ではないことを確認した.

## 検証環境

- OS: Windows 11 Home
- GPU: NVIDIA GeForce (ノート PC)
- GPU 温度: 41-46°C (サーマルスロットリング非該当)
- Python 3.13
- pochidetection v0.12.2+

---

## 1. 現象

リアルタイム推論中のフェーズ別 FPS 内訳表示 (v0.12.2 で追加) により, 以下の挙動が確認された.

- 開始後 約 3 秒間は画像推論と同等の推論速度
- その後, infer フェーズの処理時間が徐々に増加
- モデル形式 (PyTorch/ONNX/TensorRT) やモデルサイズに関係なく同様の現象
- 速度低下は永続ではなく, 一定周期で回復と低下を繰り返す

## 2. 原因調査

### 仮説と切り分け

| 仮説 | 結果 |
|------|------|
| サーマルスロットリング (GPU 過熱) | ❌ GPU 温度 41°C で非該当 |
| `torch.cuda.synchronize()` の累積影響 | ❌ 周期的な回復が説明できない |
| DirectShow フレームバッファ蓄積 | ❌ 推論時間自体の変動が説明できない |
| **GPU Power Limit によるクロックスロットリング** | ✅ **原因確定** |

### 検証: nvidia-smi によるクロック監視

```bash
nvidia-smi dmon -s pc -d 1
```

推論中の GPU クロック変動:

```
# gpu  pwr  temp    sm   mem  mclk   pclk
    0   43    46     -  10251  2790   # ← 高クロック (高速推論)
    0   68    46     -  10251  2790
    0   54    45     -   5001  1185   # ← クロック低下開始
    0   37    45     -   5001  1005
    0   36    45     -   5001   900
    0   36    45     -   5001   825
    0   35    45     -   5001   750   # ← 最低クロック (推論遅延)
    0   35    45     -    810  1200   # ← クロック回復開始
    0   46    46     -  10251  2520
    0   40    45     -  10501  2520   # ← 高クロックに復帰
```

GPU コアクロックが **2790MHz → 750MHz → 2520MHz** と大幅に変動している. 消費電力 (pwr) が 68W に達した時点で Power Limit が発動し, クロックが強制的に低下する. 温度が低くても電力制限は独立して作用する.

## 3. メカニズム

```
高クロック (2790MHz) で推論
    ↓
消費電力が Power Limit に到達 (~68W)
    ↓
GPU がクロックを強制低下 (750MHz まで)
    ↓
消費電力が低下 → Power Limit 解除
    ↓
クロック復帰 (2520MHz)
    ↓
再び高クロックで推論 → 繰り返し
```

この挙動は NVIDIA GPU Boost の仕様であり, 特にノート PC では TDP (Thermal Design Power) が低く設定されているため顕著に現れる.

## 4. 緩和策

### ハードウェア側 (効果: 大)

```bash
# 管理者権限で実行
nvidia-smi -pm 1                    # Persistence Mode 有効化
nvidia-smi -pl <max_power_limit>    # Power Limit を最大に設定
```

ただしノート PC では BIOS や電源プランで制限されており, 変更が効かない場合がある.

### ソフトウェア側 (効果: 小〜中)

- `--interval 2` 等で推論頻度を下げ, GPU 負荷を軽減
- TensorRT FP16/INT8 で推論あたりの消費電力を削減
- GPU クロックを低めに固定 (`nvidia-smi -lgc <min>,<max>`) してスロットリングを回避

### 対処不要と判断した理由

- ソフトウェアの不具合ではなく, GPU の電力管理による正常な挙動
- フェーズ別 FPS 内訳表示により, ユーザーが原因を特定できる状態になっている
- 緩和策はハードウェア構成に依存するため, フレームワーク側での自動対処は不適切

## 5. 結論

リアルタイム推論の速度低下は GPU の Power Limit によるクロックスロットリングが原因であり, pochidetection のソフトウェア不具合ではない. フェーズ別 FPS 内訳表示 (#374) により, ユーザーがボトルネックを特定できる仕組みは整っている.
