# 物体検出の評価指標ガイド

関連 Issue: #249

## 概要

物体検出モデルの精度を評価するための主要な指標を解説する.
本プロジェクトの学習・推論パイプラインが出力する混同行列, PR 曲線, F1-Confidence 曲線の読み方も併せて説明する.

---

## 1. 基本用語 (TP / FP / FN)

物体検出では, モデルの予測と正解 (Ground Truth, GT) を IoU (Intersection over Union) でマッチングし, 各予測を以下の 3 つに分類する.

### 1.1 定義

| 略称 | 名称 | 意味 |
|------|------|------|
| TP | True Positive | GT に対応する正しい検出 |
| FP | False Positive | GT に対応しない誤検出 |
| FN | False Negative | 検出されなかった GT (見逃し) |

**TN (True Negative) は物体検出では使用しない.** 背景ピクセルは事実上無限にあるため, 「正しく検出しなかった背景」を数えることに意味がないためである.

### 1.2 IoU ベースのマッチング

予測と GT のマッチングは IoU (重なり度合い) で判定する.

```
IoU = 交差領域の面積 / 和集合領域の面積
```

一般的な閾値は **IoU >= 0.5** (COCO 標準の mAP@.5 に相当). 本プロジェクトでもこの閾値を使用する.

マッチングのルール:
- 1 つの GT に対してマッチできる予測は **1 つだけ** (1:1 マッチング)
- IoU が最大かつ閾値以上のペアが優先的にマッチする
- マッチ相手がない予測 → **FP**
- マッチ相手がない GT → **FN**

### 1.3 具体例

画像内に GT が 3 個, 予測が 4 個ある場合:

| 予測 | マッチした GT | IoU | 結果 |
|------|--------------|-----|------|
| Pred_1 | GT_A | 0.82 | TP |
| Pred_2 | GT_B | 0.61 | TP |
| Pred_3 | GT_C | 0.35 | FP (IoU < 0.5) |
| Pred_4 | なし | - | FP (余剰検出) |
| - | GT_C | - | FN (マッチ不成立) |

## 2. Precision / Recall / F1 Score

### 2.1 Precision (適合率)

```
Precision = TP / (TP + FP)
```

**「検出したもののうち, 正解はどれだけか」** を表す. FP (誤検出) が多いと低下する.

### 2.2 Recall (再現率)

```
Recall = TP / (TP + FN)
```

**「正解のうち, どれだけ検出できたか」** を表す. FN (見逃し) が多いと低下する.

### 2.3 F1 Score

```
F1 = 2 * Precision * Recall / (Precision + Recall)
```

Precision と Recall の **調和平均**. 両方がバランスよく高いときに高くなる.

### 2.4 信頼度閾値との関係

モデルの出力する信頼度 (confidence score) に閾値を設定して検出をフィルタリングする:

- **閾値を高くする** → 低スコアの検出を除去 → FP が減り **Precision が上がる** が, FN が増え **Recall が下がる**
- **閾値を低くする** → 多くの検出を残す → FN が減り **Recall が上がる** が, FP が増え **Precision が下がる**

### 2.5 計算例

| クラス | TP | FP | FN | Precision | Recall | F1 |
|--------|----|----|-----|-----------|--------|------|
| dog | 13 | 5 | 0 | 0.72 | 1.00 | 0.84 |
| cat | 2 | 2 | 5 | 0.50 | 0.29 | 0.36 |
| bird | 14 | 5 | 0 | 0.74 | 1.00 | 0.85 |
| fish | 0 | 3 | 7 | 0.00 | 0.00 | 0.00 |

- dog / bird: Recall は完璧だが, 誤検出で Precision が低い
- cat: 見逃しが多く Recall が低い
- fish: 全て見逃し, かつ検出した 3 件も全て誤検出

## 3. 混同行列 (Confusion Matrix)

### 3.1 概要

混同行列は **行に GT のクラス, 列に予測クラス** を配置したマトリクスである.

```
              予測
         dog  cat  bird  BG
GT dog  [ 13    0    0    0 ]
   cat  [  0    2    0    5 ]
   bird [  0    0   14    0 ]
   BG   [  5    2    5    - ]
```

### 3.2 読み方

- **対角要素**: 正しい分類 (TP)
- **対角以外の GT 行**: クラス間の誤分類 (例: cat を dog と誤検出)
- **BG 列** (GT 側の最下行以外): FN — GT があるのに検出されなかった (見逃し)
- **BG 行** (最下行): FP — GT がないのに検出された (誤検出)

### 3.3 注意点

BG 行の FP には以下の 2 種類が混在する:
- **背景からの誤検出**: 物体がない場所を検出した
- **重複検出**: 同じ物体に対して複数の検出が出た (2 つ目以降がマッチ相手なしで FP になる)

混同行列だけではこの 2 種類を区別できないため, 重複検出が疑われる場合は NMS (Non-Maximum Suppression) 閾値や信頼度閾値の調整を検討する.

本プロジェクトの出力: `work_dirs/YYYYMMDD_XXX/best/inference_NNN/confusion_matrix.html`

## 4. PR 曲線 (Precision-Recall Curve)

### 4.1 概要

信頼度閾値を変化させたときの Precision と Recall の関係をプロットした曲線.

- **X 軸**: Recall (0.0 ~ 1.0)
- **Y 軸**: Precision (0.0 ~ 1.0)
- **右上に近いほど性能が良い** (Precision と Recall の両方が高い)

### 4.2 AP (Average Precision)

PR 曲線の下の面積 (Area Under Curve) が **AP**. クラスごとに算出する.

```
AP = PR 曲線下の面積 (0.0 ~ 1.0)
```

### 4.3 mAP (mean Average Precision)

全クラスの AP の平均値. モデル全体の検出性能を 1 つの数値で表す.

```
mAP = 全クラスの AP の平均
```

### 4.4 本プロジェクトでの利用

- torchmetrics の `MeanAveragePrecision` で計算
- クラス別 + 全クラス平均の PR 曲線を Plotly で可視化
- 出力: `work_dirs/YYYYMMDD_XXX/pr_curve.html`

## 5. F1-Confidence 曲線

### 5.1 概要

信頼度閾値を変化させたときの F1 Score の変化をプロットした曲線.

- **X 軸**: Confidence (0.0 ~ 1.0)
- **Y 軸**: F1 Score
- **F1 が最大になる閾値** が, そのクラスの最適な信頼度閾値

### 5.2 PR 曲線との違い

PR 曲線では信頼度の具体的な値が分からない. F1-Confidence 曲線は **「信頼度をいくつに設定すれば最も良いバランスになるか」** を直接示す.

### 5.3 読み方

- 曲線のピーク位置 = 最適閾値
- ピークが左側 (低い信頼度) にある → モデルの出力スコアが全体的に低い
- ピークが右側 (高い信頼度) にある → モデルの出力スコアが全体的に高い
- クラス別の曲線を比較することで, クラスごとの検出難易度の差が分かる

### 5.4 本プロジェクトでの利用

- `F1ConfidencePlotter` で torchmetrics の `precision` + `scores` テンソルから算出
- クラス別 + 全クラス平均の曲線を Plotly で可視化
- 最大 F1 の閾値をマーカーで表示
- 出力: `work_dirs/YYYYMMDD_XXX/f1_confidence.html`

## 6. NMS (Non-Maximum Suppression)

精度評価に直接影響するため, 補足として記載する.

### 6.1 概要

NMS は重複した検出ボックスを除去するアルゴリズム. 信頼度スコアが高い検出を優先的に残し, それと大きく重なる (IoU が閾値以上の) 検出を抑制する.

### 6.2 クラス別 NMS vs クラス非依存 NMS

| 方式 | 動作 |
|------|------|
| クラス別 (class-aware) | 同じクラス同士でのみ抑制 |
| クラス非依存 (class-agnostic) | クラスに関係なく抑制 |

### 6.3 精度評価への影響

NMS の IoU 閾値が低すぎると正当な検出も抑制され FN が増加する. 高すぎると重複検出が残り FP が増加する. F1-Confidence 曲線と組み合わせて最適な閾値を探索するのが効果的.
